Hexo博客搭建及使用

发表于 2022-01-02 更新于 2023-04-01 分类于 Blog

前期准备

安装 Node.js

  • 官网下载并安装：<https://nodejs.org/zh-cn/>
  • 将Node.js添加到环境变量
  • 验证：node -v

安装 Git

  • 官网下载并安装：<https://git-scm.com/>
  • 将 Git 添加到环境变量（安装时默认添加）
  • 验证：git -v

安装 Hexo

  • 打开 git bash，输入命令：npm install hexo-cli -g 全局安装 Hexo
  • 验证：hexo -v

Git 配置及 GitHub 仓库建立

  • 设置 Git 用户名及邮箱
    • git config --global user.name "${username}" ${username} 替换为 GitHub 用户名
    • git config --global user.email "${email}" ${email} 替换为注册 GitHub 的邮箱
    • 查看全局配置是否成功 git config --global --list
  • 设置 ssh key
    • ssh-keygen -t ed25519 -C "your_email@example.com" 一路确认生成 ssh key
    • ls -al ~/.ssh 查看是否成功生成，复制 id_ed25519.pub 文件中的内容到 GitHub 设置中的 ‘SSH and GPG keys’ 项中
    • 验证：ssh -T git@github.com
  • 新建 GitHub 仓库
    • 新建一个仓库，仓库名为 ‘github用户名.github.io’

使用 Hexo 搭建博客

本地初始化

  • hexo init blog 初始化一个 blog 文件夹放置 hexo 代码
  • cd blog 进入 blog 文件夹
  • hexo g 生成静态文件
  • hexo s 启动本地服务器，默认网址为：<http://localhost:4000/>

Hexo 博客使用

Hexo 官网：<https://hexo.io/zh-cn/>

写文章

hexo new [layout] <title> 命令用于新建文章或页面

layout 可取 post, page, draft

布局 路径
post source/_posts
page source
draft source/_drafts

草稿

草稿在建立时会被保存到 source/_drafts 文件夹，可以通过 publish 命令将草稿移动到 source/_posts 文件夹

草稿默认不会显示在页面中，可以在执行时加上 --draft 参数（如 hexo s --draft）或将站点配置文件 _config.yml 中的 render_drafts 参数设为 true 也可以预览草稿

模板

在使用 hexo new 命令新建文章时，会根据 scaffolds 文件夹内对应的文件来建立文件

如 scaffolds/post.md 中的内容会在新建 post 时自动添加到文章中

分类及标签

只有文章支持分类与标签，可以在 Front-matter 中设置。在 Hexo 中，分类具有顺序性和层次性，也就是说 Foo, Bar 不等于 Bar, Foo；而标签没有顺序和层次
